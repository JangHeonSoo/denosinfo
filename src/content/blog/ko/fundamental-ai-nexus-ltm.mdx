<strong>ChatGPT</strong>에게 엑셀 파일 분석을 맡겨본 적 있는가? 매출 데이터 넣고 "다음 분기 예측해줘" 했더니 돌아온 답변이 뭔가… 아쉬웠을 거다. 

엉뚱한 행을 참조하거나, 열 간의 관계를 완전히 무시하거나. 그건 당신 프롬프트가 나빠서가 아니다. <strong>LLM</strong>은 태생적으로 표 데이터를 잘 못 다룬다.

2026년 2월 5일, <strong>DeepMind</strong> 출신 팀이 만든 스타트업 <strong>Fundamental AI</strong>가 스텔스 모드에서 나왔다. 들고 나온 건 시리즈 A 2.55억 달러(약 3,700억 원)와 <strong>NEXUS</strong>라는 이름의 모델. 그런데 이 모델, <strong>LLM</strong>이 아니다.

<br/>

## LLM이 표 데이터 앞에서 무력한 이유

<strong>GPT</strong>든 <strong>Claude</strong>든, <strong>트랜스포머</strong> 기반 모델은 텍스트 시퀀스를 처리하도록 설계됐다. 왼쪽에서 오른쪽으로 읽고, 토큰 간의 관계를 파악한다. 소설이나 코드에는 딱이다.

그런데 표 데이터는 다르다. 100개 열, 50만 행짜리 <strong>엔터프라이즈 데이터셋</strong>을 생각해보자. 행과 열 사이에 비선형적 관계가 얽혀 있고, 순서가 의미를 갖지 않는다. <strong>LLM</strong>에게 이걸 넘기면? 마치 영어 사전으로 수학 문제를 푸는 격이다.

기존 방식도 문제가 많았다. 기업들은 <strong>XGBoost</strong>나 <strong>Random Forest</strong> 같은 전통 ML 알고리즘을 썼는데, 이건 유스케이스마다 데이터 사이언티스트가 몇 달씩 붙어서 <strong>피처 엔지니어링</strong>을 해야 한다. 고객 이탈 예측 모델 하나, 수요 예측 모델 하나, 가격 최적화 모델 하나… 끝이 없다.

<br/>

## NEXUS: 표 데이터 전용 파운데이션 모델

![NEXUS 모델의 구조적이고 체계적인 데이터 처리 아키텍처 시각화](/images/nexus_architecture_structure_1770592503215.png)

<strong>Fundamental</strong>이 들고 나온 <strong>NEXUS</strong>는 <strong>Large Tabular Model(LTM)</strong>이라는 새로운 카테고리를 표방한다. 핵심 특징 몇 가지가 눈에 띈다.

<strong>트랜스포머를 쓰지 않는다.</strong> <strong>OpenAI</strong>, <strong>Anthropic</strong>, <strong>Google</strong> — 거의 모든 AI 기업이 트랜스포머 아키텍처에 기반하는데, <strong>Fundamental</strong>은 완전히 다른 길을 갔다. 표 데이터의 비순차적 특성에 맞는 자체 아키텍처를 개발했다.

<strong>결정론적(Deterministic) 모델이다.</strong> 같은 데이터를 넣으면 매번 같은 결과가 나온다. <strong>LLM</strong>의 확률적 특성이 만드는 "매번 다른 답변" 문제가 없다. 금융이나 의료처럼 일관된 예측이 필수인 영역에서 이건 어마어마한 장점이다.

<strong>코드 한 줄이면 된다.</strong> 기존 데이터 스택에 연결하면, 수동 <strong>피처 엔지니어링</strong> 없이 원시 데이터에서 바로 패턴과 의존성을 학습한다. CEO <strong>Jeremy Fraenkel</strong>의 말을 빌리면, "데이터 사이언티스트 팀 전체가 몇 달간 작업한 것보다 나은 성능"을 낸다고.

<br/>

## 숫자가 말해주는 것들

![비즈니스 성장과 데이터의 신뢰성을 보여주는 홀로그래픽 차트](/images/nexus_business_growth_1770592523708.png)

기업가치 12억 달러. 시리즈 A 치고는 파격적이다. 시드 3,000만 달러에 시리즈 A 2.25억 달러를 합쳐 총 2.55억 달러를 확보했다.

공개 론칭 전부터 <strong>포춘 100</strong> 기업 여러 곳과 7자릿수(100만 달러 이상) 계약을 체결했다. 수요 예측, 가격 예측, 고객 이탈 분석 등에 쓰이고 있다. <strong>AWS</strong>와도 파트너십을 맺어 <strong>Amazon SageMaker HyperPod</strong>에서 학습하고, AWS 대시보드를 통해 바로 배포할 수 있다.

<br/>

## 왜 지금인가

사실 "표 데이터를 잘 분석하자"는 목표가 새로운 건 아니다. 그런데 왜 지금 이 시점에 이 회사가 주목받는 걸까.

첫째, <strong>LLM</strong> 피로감이 쌓이고 있다. 기업들이 <strong>ChatGPT</strong>를 도입했는데, 정작 가장 많이 쓰는 데이터는 스프레드시트와 데이터베이스다. <strong>LLM</strong>으로는 이걸 제대로 처리할 수 없다는 걸 1~2년 써보고 깨달은 셈이다.

둘째, 예측 가능성에 대한 수요가 폭발했다. 확률적 답변이 아니라 <strong>결정론적 예측</strong>을 원하는 기업이 많다. 특히 금융, 보험, 제조 같은 규제 산업에서.

셋째, 인프라가 준비됐다. <strong>AWS</strong>, <strong>GCP</strong> 같은 클라우드 플랫폼이 대규모 모델 학습과 배포를 손쉽게 만들어놨다. <strong>Fundamental</strong>은 이 위에 올라탈 수 있었다.

<br/>

## 개발자에게 의미하는 것

만약 당신이 데이터 파이프라인을 만들거나, <strong>ML 모델</strong>을 서빙하는 일을 한다면, <strong>NEXUS</strong> 같은 도구의 등장은 꽤 큰 변화를 예고한다.

<strong>피처 엔지니어링</strong>에 쏟던 시간이 줄어든다. 원시 데이터를 바로 넘기면 되니까. 유스케이스마다 별도 모델을 만들 필요도 줄어든다. 하나의 <strong>파운데이션 모델</strong>이 다양한 예측 작업을 처리한다.

물론, 이게 데이터 사이언티스트의 종말을 의미하진 않는다. 데이터 품질 관리, 비즈니스 컨텍스트 해석, 결과 검증은 여전히 사람의 몫이다. 다만, "삽질"의 비중이 줄고 "판단"의 비중이 늘어나는 방향인 건 확실하다.

<br/>

## 한 발짝 떨어져서 보면

AI 업계가 "더 큰 <strong>LLM</strong>, 더 긴 컨텍스트 윈도우"를 외치는 동안, <strong>Fundamental</strong>은 아예 다른 판을 깔았다. 시 쓰는 AI 말고, 엑셀을 진짜로 이해하는 AI를 만들겠다는 거다.

솔직히, 이 접근이 맞다고 본다. 기업 데이터의 80%는 표 형태다. 여기서 가치를 뽑아내는 도구가 <strong>LLM</strong>이 아니었다면, 전용 모델이 나오는 건 시간문제였다.

<strong>NEXUS</strong>가 약속대로 작동하는지는 더 지켜봐야 한다. 하지만 "<strong>LLM</strong>만이 답"이라는 고정관념에 균열을 내는 건, 그것만으로도 충분히 가치 있는 일이다.
